# 服务器端数据采集命令

当前 Codex 本地环境无法直接连上 OpenEuler 跳板服务器：

```text
kex_exchange_identification: Connection closed by remote host
Connection closed by 10.137.144.91 port 9001
```

所以服务器端数据需要你在已登录的服务器终端里执行下面命令。执行后把 `test.o`、`test.e` 或截图发回来即可。

## 1. SVD 选题：OpenEuler 服务器

进入 SVD 目录：

```bash
cd ~/svd
```

确认目录里有这些文件：

```bash
ls
```

应能看到：

```text
README.md
main.cpp
bidiagonalization.cpp
gkh.cpp
test.sh
qsub.sh
```

正式运行 O2 / 4 核数据：

```bash
bash test.sh 1 1 4 -O O2 -s 20260410
```

如果老师指定的实验编号不是 `1`，把第一个 `1` 替换成指定编号。

查看任务：

```bash
qstat
qstat -n
```

任务结束后保存输出：

```bash
cp test.o test_O2_4cores_final.o
cp test.e test_O2_4cores_final.e
cat test.o
cat test.e
```

需要截图的位置：

```text
1. bash test.sh 1 1 4 -O O2 -s 20260410 命令
2. qstat 或 qstat -n 中任务状态
3. cat test.o 中每个 case 的 PASS
4. cat test.o 最后的 “通过: 5 / 5”
5. cat test.o 中总上二对角化耗时和总 GKH 迭代耗时
6. cat test.e，如果只有 Authorized users only 可作为无关键错误证明
```

## 2. 可选：不同优化等级数据

如果报告需要补充优化等级对比，按顺序跑：

```bash
for opt in O0 O1 O2 O3 Ofast; do
  bash test.sh 1 1 4 -O $opt -s 20260410
  cp test.o test_${opt}_4cores_final.o
  cp test.e test_${opt}_4cores_final.e
done
```

跑完后打包输出：

```bash
tar -czf svd_outputs_final.tar.gz test_*_4cores_final.o test_*_4cores_final.e
ls -lh svd_outputs_final.tar.gz
```
